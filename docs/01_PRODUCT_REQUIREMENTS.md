# 01. Product Requirements

## Цель продукта

QuoteFlow должен позволить пользователю подготовить простую смету и коммерческое предложение: описать проект, добавить позиции, применить скидки и пользовательскую налоговую ставку, проверить итоговую сумму, сохранить черновик и подготовить JSON, CSV или PDF.

QuoteFlow не является бухгалтерской программой и не формирует юридически значимые счета. Налоговая ставка задаётся пользователем и не позиционируется как актуальное требование законодательства.

## Целевая аудитория

- малый бизнес;
- независимые эксперты;
- небольшие команды;
- посетители портфолио Stalar Vision, оценивающие формат технической реализации.

## Основной пользовательский сценарий

1. Создать новый расчёт.
2. Указать название проекта и условные данные клиента.
3. Добавить, изменить или удалить позиции.
4. Для позиции указать название, описание, количество, единицу измерения, цену и скидку.
5. Увидеть line totals и subtotal.
6. Применить общую скидку.
7. Выбрать «Без налога» или задать произвольную налоговую ставку.
8. Увидеть налог и итоговую сумму.
9. Сохранить, открыть или удалить локальный черновик.
10. Импортировать или экспортировать JSON.
11. Экспортировать позиции в CSV.
12. Получить backend preview.
13. Сформировать print-friendly representation и PDF.
14. Увидеть демонстрационный payload, пригодный для будущей API-интеграции, без фактической внешней отправки.

## MVP scope

- динамические позиции;
- автоматический точный пересчёт;
- контролируемое округление;
- понятная frontend- и backend-валидация;
- сохранение, загрузка, удаление и полная очистка черновиков;
- JSON import/export с `schemaVersion`;
- CSV export;
- stateless REST API;
- PDF generation с русским текстом;
- print-friendly view;
- responsive UI без горизонтального скролла;
- keyboard navigation, labels, focus states и базовая accessibility;
- empty, error и loading states;
- synthetic fixtures;
- frontend unit tests расчётного ядра;
- backend tests;
- `.env.example`;
- Docker-based local launch;
- README, API и security documentation.

## Out of scope

- регистрация и пользовательские аккаунты;
- платежи и тарифы;
- CRM/ERP;
- server-side draft storage;
- произвольные webhook URL;
- реальные email, Telegram или webhook delivery;
- AI features;
- загрузка файлов;
- многоарендность;
- административная панель;
- сложная аналитика;
- юридическое, бухгалтерское или налоговое сопровождение.

## Validation states

Нужно различать:

- **draft** — может содержать пустые или промежуточные UI-значения и сохраняться локально;
- **valid calculation input** — проходит строгую схему и может быть рассчитан, экспортирован или отправлен на preview;
- **valid document input** — дополнительно имеет обязательные данные документа и может быть отправлен на PDF generation.

## Приватность и предупреждения

В UI, README и PDF должна присутствовать честная демонстрационная рамка. Пользователь должен видеть, что:

- нельзя вводить реальные персональные и конфиденциальные данные;
- `localStorage` не является надёжным долговременным хранилищем;
- расчёты необходимо проверять;
- налоговая ставка задаётся пользователем;
- приложение не заменяет бухгалтера или юриста.

## Acceptance criteria MVP

MVP готов к review, когда формула и округление воспроизводимы; позиции и черновики управляются; import/export работает; PDF поддерживает русский текст и несколько страниц; frontend/backend и Docker Compose запускаются локально; тесты и production build проходят; секретов и реальных данных нет; README соответствует реализации; production deployment ещё не выполнялся.
