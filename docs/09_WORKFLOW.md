# 09. Workflow

## Язык взаимодействия

- Общение с владельцем, объяснения, анализ, review и стратегические решения выполняются на русском языке.
- Готовые prompts для Codex/Copilot пишутся на английском языке.
- Имена файлов, путей, типов, endpoints, переменных и других технических сущностей сохраняются в исходной форме.

## Роли

### GPT

- определяет один следующий логичный этап;
- выбирает самый экономный исполнитель: GPT, владелец локально или Codex;
- готовит точный английский prompt для Codex только когда задача действительно требует локальной code implementation;
- читает актуальные файлы из GitHub после push;
- проводит code, product и security review;
- может создавать, редактировать и удалять любые доступные GitHub files в пределах явно согласованного scope;
- не меняет remote code/config при наличии конфликтующих незакоммиченных local changes;
- поддерживает roadmap и handoff в соответствии с фактической реализацией.

### Codex / Copilot

- изменяет только код, tests, fixtures, assets и технические конфигурации, прямо относящиеся к задаче;
- не изменяет `README.md` и `docs/**`;
- не создаёт commit;
- не выполняет push, merge или rebase;
- не меняет Git history;
- не выполняет production deployment;
- запускает указанные проверки;
- возвращает полный diff, результаты проверок и `git status --short`.

### Владелец

- выполняет согласованные локальные команды, validation и Git operations;
- передаёт prompt Codex, когда выбран Codex workflow;
- проверяет результат и список файлов;
- после review выполняет commit и push, если GitHub write не выполняет GPT;
- сообщает GPT commit SHA;
- синхронизирует local `main` после удалённых commits GPT.

## Рабочий цикл

1. GPT анализирует актуальный GitHub state и выбирает минимально затратный безопасный workflow.
2. Рутинные GitHub/docs/review задачи выполняет GPT; локальные команды и проверки — владелец.
3. Для сложной code implementation GPT формирует один сфокусированный английский prompt для Codex с подходящими model/reasoning parameters.
4. Codex возвращает diff, проверки и status без commit/push, если не дано отдельное разрешение.
5. GPT проводит review; владелец выполняет локальный commit/push либо GPT делает согласованное GitHub write.
6. GPT проверяет фактический commit и CI, затем обновляет документацию.
7. Владелец синхронизирует local `main` через `git pull --ff-only origin main`.
8. Только после этого начинается следующий этап.

## Постоянные ограничения

- QuoteFlow остаётся отдельным проектом и не изменяет `site-stalarvision`.
- Нельзя придумывать клиентов, отзывы, результаты или метрики.
- Нельзя использовать реальные персональные или конфиденциальные данные.
- Репозиторий публичный с момента создания; проект использует MIT License.
- Production deployment всегда требует отдельного контролируемого решения после resource/security review.
- Новые зависимости добавляются только в явно согласованном scope.
- При dirty local working tree удалённые изменения пересекающихся code/config files не выполняются.
- Codex не используется для действий, которые GPT или владелец могут безопасно выполнить без снижения качества.
