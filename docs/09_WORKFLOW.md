# 09. Workflow

## Язык взаимодействия

- Общение с владельцем, объяснения, анализ, review и стратегические решения выполняются на русском языке.
- Готовые prompts для Codex/Copilot пишутся на английском языке.
- Имена файлов, путей, типов, endpoints, переменных и других технических сущностей сохраняются в исходной форме.

## Роли

### GPT

- определяет один следующий логичный этап;
- готовит точный английский prompt для Codex;
- читает актуальный код из GitHub после push;
- проводит code, product и security review;
- самостоятельно создаёт, редактирует и удаляет `README.md` и `docs/**` непосредственно в GitHub;
- поддерживает roadmap и handoff в соответствии с фактической реализацией.

### Codex / Copilot

- изменяет только код, tests, fixtures, assets и технические конфигурации, прямо относящиеся к задаче;
- не изменяет `README.md` и `docs/**`;
- не создаёт commit;
- не выполняет push, merge или rebase;
- не меняет Git history;
- не выполняет production deployment;
- запускает указанные проверки;
- возвращает полный diff, результаты проверок и `git status --short`.

### Владелец

- передаёт prompt Codex;
- проверяет результат и список файлов;
- после review выполняет commit и push;
- сообщает GPT commit SHA;
- выполняет локальный `git pull --ff-only origin main` после документационных commits GPT.

## Рабочий цикл

1. GPT анализирует актуальный GitHub state.
2. GPT формирует один английский prompt для Codex.
3. Codex изменяет код, но не документацию.
4. Codex возвращает diff, проверки и status.
5. GPT проводит предварительный review результата.
6. Владелец выполняет commit и push разрешённых файлов.
7. GPT читает commit из GitHub и проводит окончательный review.
8. GPT самостоятельно обновляет документацию в GitHub.
9. Владелец синхронизирует локальный main через `git pull --ff-only origin main`.
10. Только после этого начинается следующий этап.

## Постоянные ограничения

- QuoteFlow остаётся отдельным проектом и не изменяет `site-stalarvision`.
- Нельзя придумывать клиентов, отзывы, результаты или метрики.
- Нельзя использовать реальные персональные или конфиденциальные данные.
- Public GitHub, лицензия и production deployment требуют отдельного решения.
- Новые зависимости добавляются только в явно согласованном scope.
