# 06. Security

QuoteFlow не должен описываться как полностью безопасный. Этот документ фиксирует boundaries и обязательные требования перед публичным размещением.

## Threat model MVP

Основные риски:

- ввод реальных персональных или конфиденциальных данных в public demo;
- импорт вредоносного или неожиданного JSON;
- слишком большие request body;
- попытка внедрить HTML или PDF markup через текстовые поля;
- path traversal при работе с именами файлов;
- утечка пользовательского содержимого через логи;
- случайное добавление секретов в Git history;
- небезопасная CORS-конфигурация перед public deployment.

## Secrets and configuration

- Секреты запрещено хранить в Git.
- Будущий `.env` не должен попадать в repository.
- Будущий `.env.example` должен содержать только безопасные placeholder values.
- Перед публикацией нужно проверить Git history на случайно добавленные секреты.

## Backend validation

Backend должен:

- ограничивать JSON request size;
- ограничивать количество позиций;
- проверять длину строк;
- запрещать отрицательные денежные значения;
- проверять диапазон basis points;
- проверять формат `quantity`;
- игнорировать или отклонять неизвестные поля импортируемой модели по явному правилу;
- пересчитывать totals самостоятельно.

## Logging

Нельзя логировать request body с пользовательским содержимым. Логи не должны содержать токены, реальные персональные данные или конфиденциальные данные. `requestId` должен быть техническим идентификатором и не должен кодировать пользовательскую информацию.

## Browser storage

`localStorage` используется только для draft storage. Ограничения:

- данные доступны в браузере пользователя;
- данные могут быть очищены браузером;
- данные не являются надежным долгосрочным хранилищем;
- public demo должен предупреждать не вводить реальные персональные или конфиденциальные данные.

## JSON import

Импортируемый JSON должен обрабатываться как недоверенный input. Неизвестные поля не должны молча попадать в доверенную модель. Ошибки import должны быть понятными и не должны раскрывать внутренние детали реализации.

## PDF boundaries

Текстовые поля должны обрабатываться как данные, а не как HTML. PDF generation должна экранировать или безопасно размещать пользовательский текст. Имя скачиваемого файла должно формироваться безопасно и не должно позволять path traversal.

Если PDF generation использует временные файлы, они должны очищаться после запроса. Предпочтительно формировать PDF в памяти.

## Network boundaries

В MVP отсутствуют:

- произвольные внешние URL;
- реальная отправка email;
- реальная отправка Telegram;
- real webhook delivery;
- скрытая отправка данных во внешние сервисы.

## CORS and public deployment

Перед public deployment требуется CORS allowlist. Rate limiting является обязательным требованием перед public deployment. Security headers являются требованием production-инфраструктуры.

## Dependencies

Dependency audit должен выполняться без автоматического `force fix`. Исправления зависимостей требуют review, чтобы не вносить несовместимые изменения незаметно.

## Fixtures

Fixtures должны быть synthetic. В них нельзя добавлять реальные персональные, коммерческие или конфиденциальные данные.
